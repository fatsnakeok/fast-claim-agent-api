MVP3 智能核保 — 状态机图

Quote 报价单状态变迁 + Policy 保单生命周期

Quote 报价单 — 状态变迁 核心实体 
stateDiagram-v2
    direction LR

    [*] --> APPROVED : riskScore ≤ 60\nLowRiskQuote.execute()\n创建 Quote + 持久化
    [*] --> REFERRED : 60 < riskScore < 80\nMediumRiskReview.execute()\n创建 Quote + 持久化
    [*] --> DECLINED : riskScore ≥ 80\nHighRiskDecline.execute()\n创建 Quote + 持久化(含拒绝原因)

    REFERRED --> APPROVED : 核保员人工审批\nAgentService.approveQuote()\n可选覆盖 premiumAmount

    APPROVED --> [*] : PaymentService.pay()\n支付成功 → 创建 Policy\nQuote 状态保持不变

    note right of APPROVED
      可支付状态
      有效期 30 天
      过期后不可支付
    end note

    note right of REFERRED
      待人工审核
      有效期 30 天
      过期后不可审批
    end note

    note right of DECLINED
      终态，不可逆转
      含 rejectionReason 字段
      记录各因子扣分明细
    end note
Policy 保单 — 生命周期 实体状态 
stateDiagram-v2
    direction LR

    [*] --> ACTIVE : PaymentService.pay()\n创建 Policy
    ACTIVE --> EXPIRED : expirationDate 到达\n系统自动判断
    ACTIVE --> CANCELLED : 用户退保(预留)\n本期未实现

    note right of ACTIVE
      policyNumber: POL-{ts}-{6位随机}
      effectiveDate: 签发时间
      expirationDate: +lengthOfYear()
      coverageType 来自 Quote
      premiumAmount 来自 Quote
    end note
端到端状态流转（Quote + Policy 关联视图） 概览 
stateDiagram-v2
    state 用户发起 {
        [*] --> 核保申请
    }

    state UnderwritingAgent_核保 {
        核保申请 --> LLM提取车辆信息
        LLM提取车辆信息 --> 客户车辆查找
        客户车辆查找 --> 6因子风险评估
        6因子风险评估 --> 保费计算
        保费计算 --> 阈值判定
    }

    state Quote_报价单 {
        阈值判定 --> APPROVED : ≤60分
        阈值判定 --> REFERRED : 61~79分
        阈值判定 --> DECLINED : ≥80分

        REFERRED --> APPROVED : 人工审批
    }

    state Policy_保单 {
        APPROVED --> ACTIVE : 支付签单
        ACTIVE --> [*] : 到期
    }

    state 错误路径 {
        阈值判定 --> ERROR : 客户不存在
        阈值判定 --> ERROR : 车辆未找到
        阈值判定 --> ERROR : LLM提取失败
    }
