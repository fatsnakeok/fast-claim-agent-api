# fast-claim-agent-api — 技术选型与项目搭建指南

## 1. 技术选型

### 1.1 核心依赖

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 语言 | Java | 21 | LTS 版本，虚拟线程支持，Record 类型简化 DTO |
| 框架 | Spring Boot | 3.4.0 | 成熟生态，与 Embabel 框架深度集成 |
| AI Agent | Embabel Agent | 0.3.5 | 为 Spring Boot 设计的 Agent 框架，@Agent 注解声明式智能体、@State 分类路由、UTILITY 规划器 |
| LLM | DeepSeek | chat / reasoner | 高性能低成本，支持中文，通过 Embabel starter 零配置接入 |
| RAG 引擎 | Embabel Lucene RAG | 0.3.5 | 纯 BM25 文本检索，零外部依赖，适合快速启动和低资源场景 |
| 文档解析 | Embabel RAG Tika | 0.3.5 | Apache Tika 集成，支持 Markdown/PDF/Word 等多种格式 |
| 数据库 | H2 | — | 内存数据库，开发/测试零配置；可随时替换为 PostgreSQL/MySQL |
| ORM | Spring Data JPA + Hibernate | — | 自动化 CRUD，ddl-auto=update 开发期免手动建表 |
| 安全 | Spring Security | — | HTTP Basic 认证 + 方法级 @PreAuthorize + RoleHierarchy |
| API 文档 | SpringDoc OpenAPI | 2.7.0 | 自动生成 Swagger UI，支持 JSR-303 校验注解 |
| Web | Spring Web + WebFlux | — | REST API + Embabel Netty 通信（Agent Shell 模式已关闭） |
| 可观测性 | Embabel Observability + Langfuse OTel | 0.3.5 / 0.4.0 | OpenTelemetry trace 导出，默认关闭，按需开启 |
| 缓存 | Spring Cache | — | ConcurrentMapCacheManager 内存缓存，零外部依赖 |
| 测试 | Embabel Agent Test + H2 | 0.3.5 | FakeOperationContext 模拟 LLM，H2 隔离测试 |
| 构建 | Maven Wrapper | — | 无需预装 Maven，`./mvnw` 即可构建 |

### 1.2 Embabel 框架能力矩阵

| 特性 | 用途 | 本项目的使用 |
|------|------|------------|
| `@Agent` | 声明一个 Java 类为智能体 | ClaimsAgent, ChatbotAgent 等 |
| `@Action` | 标记智能体中的一个步骤（LLM 调用或纯逻辑） | verifyPolicy, extractClaimInfo 等 |
| `@AchievesGoal` | 标记终态动作，成功返回后计划终止 | 所有 @State 的 handleXxx() 方法 |
| `@State` | sealed interface 的 permit 子类 → 规划器自动路由 | AutoApproved, PolicyError 等 |
| `PlannerType.UTILITY` | 强制框架按 @Action 返回类型推导执行路径 | ChatbotAgent |
| `PlannerType` 默认 | 框架根据参数/返回类型自动推导数据流 | ClaimsAgent |
| `OperationContext` | 操作上下文，提供 `ai()` LLM 调用入口和 `bind()`/`get()` Blackboard 读写 | 所有 @Action 方法 |
| `ToolishRag` | 将 RAG 搜索包装为 LLM 可调用的工具，实现 Agentic RAG | ChatbotAgent.answerQuestion |
| `UserInputGuardRail` | 框架接口，在 LLM 调用前校验用户输入 | 用户输入护栏 |
| `AssistantMessageGuardRail` | 框架接口，校验 LLM 生成的回复 | LLM 回复护栏 |
| `StuckHandler` | 卡住时的诊断回调 | 所有 Agent 均实现此接口 |
| `Blackboard` | Agent 进程内的共享状态存储 | 错误信息传播（非异常路径） |
| `AgentPlatform` | 运行时查找/创建/运行 Agent | AgentService 中编排 Agent 进程 |
| `AgentInvocation` | 简化版 Agent 调用 API | ChatService 中调用 ChatbotAgent |

### 1.3 LLM 模型分层策略

```
fast (deepseek-chat)     → 简单查询、信息提取、RAG 检索
balanced (deepseek-chat) → 聊天、理赔处理、摘要
powerful (deepseek-reasoner) → 复杂推理、多步骤决策
auto (框架默认)          → 框架自行判断
```

### 4.2 Agent 定义模式

```java
@Agent(
    description = "Agent 描述",
    planner = PlannerType.UTILITY   // 或省略使用默认数据流推导
)
@Component
public class XxxAgent implements StuckHandler {

    // ── 公共类型定义 ──
    public record XxxInput(...) {}
    public record XxxResult(...) {}

    // ── @State 密封接口 ──
    @State
    public sealed interface Decision permits StateA, StateB, ErrorState {}

    // ── 流水线 @Action ──
    @Action(description = "步骤1描述")
    public TypeA step1(UserInput userInput, OperationContext context) { ... }

    @Action(description = "步骤2描述")
    public TypeB step2(TypeA input, OperationContext context) { ... }

    // ── 入口 @Action → 分类到 @State ──
    @Action(description = "分类路由")
    public Decision classify(TypeB input, OperationContext context) {
        if (errorCondition) return new ErrorState(...);
        if (score < threshold) return new StateA(...);
        return new StateB(...);
    }

    // ── @State 实现 @AchievesGoal ──
    @State
    public record StateA(...) implements Decision {
        @AchievesGoal(description = "终态A", value = 1.0)
        @Action(description = "处理A")
        public XxxResult handle() { ... }
    }

    @State
    public record ErrorState(String message) implements Decision {
        @AchievesGoal(description = "错误终态", value = 1.0)
        @Action(description = "返回错误")
        public XxxResult handleError(OperationContext context) {
            context.bind("xxx_error", message);   // 错误写入 Blackboard
            return new XxxResult("ERROR", message);
        }
    }

    // ── StuckHandler ──
    @Override
    public StuckHandlerResult handleStuck(AgentProcess process) {
        log.error("=== STUCK: status={}", process.getStatus());
        return new StuckHandlerResult("...", this, StuckHandlingResultCode.NO_RESOLUTION, process);
    }
}
```

### 4.3 AgentService 编排模式

```java
@Service
public class AgentService {
    private static final long PROCESS_TIMEOUT_SECONDS = 120;
    private final AgentPlatform agentPlatform;

    public ResultType process(String userInput) {
        UserInput input = new UserInput(userInput);

        // 1. 查找 Agent
        Agent agent = agentPlatform.agents().stream()
            .filter(a -> a.getName().contains("Xxx"))
            .findFirst()
            .orElseThrow(() -> new RuntimeException("Agent not found"));

        // 2. 创建并运行 AgentProcess（带超时保护）
        AgentProcess process = agentPlatform.createAgentProcessFrom(agent, options, input);
        AgentProcess completed = CompletableFuture
            .supplyAsync(process::run)
            .get(PROCESS_TIMEOUT_SECONDS, TimeUnit.SECONDS);

        // 3. 提取结果
        ResultType result = completed.last(ResultType.class);
        if (result != null) return result;

        // 4. 回退：从 Blackboard 读取错误信息
        String error = (String) completed.get("xxx_error");
        if (error != null) return errorResult(error);

        throw new RuntimeException("Agent did not produce result");
    }
}
```

### 4.4 ChatbotAgent + Agentic RAG 模式

```java
@Agent(description = "客服 Agent", planner = PlannerType.UTILITY)
@Component
public class ChatbotAgent implements StuckHandler {

    private final ToolishRag rag;   // 注入 RagConfiguration 创建的 Bean

    @AchievesGoal(description = "answer_question", value = 1.0)
    @Action(description = "Agentic RAG 回答")
    public ChatOutput answerQuestion(UserInput userInput, OperationContext context) {
        String answer = context.ai()
            .withReference(rag)     // 关键：注册 RAG 为 LLM 工具
            .generateText(prompt);
        return new ChatOutput(answer);
    }
}
```

## 6. 关键决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 规划器类型 | ChatbotAgent 用 UTILITY，ClaimsAgent 用默认 | UTILITY 确保参数类型匹配不丢；默认自动推导更灵活 |
| 错误传播 | Blackboard + @State 错误路由，不抛异常 | 框架重试机制会吞掉异常，Blackboard 保证错误可达 |
| Sentinel 对象 | 查不到时返回 sentinel 而非 null | 避免 UTILITY 规划器因类型缺失 STUCK |
| LLM 超时 | 120s | DeepSeek API 的合理上限，超时触发 StuckHandler |
| RAG 模式 | Agentic RAG（ToolishRag），非传统检索注入 | LLM 自主决定检索时机和查询词，多轮迭代更灵活 |
| 缓存策略 | ConcurrentMapCacheManager（内存）+ TTL 5min | 开发期零外部依赖，生产可替换为 Redis |
| 数据库 | H2 内存模式 + ddl-auto=update | 开发期自动建表零配置，生产改 PostgreSQL + validate |
| 安全 | HTTP Basic + InMemoryUserDetailsManager | 开发/演示用，生产改 OAuth2/JWT + JDBC |
| API 文档 | SpringDoc OpenAPI 自动扫描 | 无需手写文档，注解即文档 |
