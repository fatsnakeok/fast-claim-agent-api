# 智能保险平台 — 业务梳理思维脑图

> 按业务域拆分为 6 个独立脑图，图片见 `doc/diagrams/mindmap-0*.png`。

## 1. 角色体系

> 图片：`doc/diagrams/mindmap-01-角色体系.png`

```
角色体系
├── 普通用户 — 投保申请、查看保单、发起理赔、支付保费、咨询客服
├── 核保员 — 审核中风险案件(61-79分)、审批REFERRED报价单、可调整保费
├── 理赔员 — 审核存疑案件(30-69分)、批准赔付(≤保费×5)、拒绝并记录原因
└── 管理员 — 管理知识库文档、全部操作权限
```

## 2. 投保域

> 图片：`doc/diagrams/mindmap-02-投保域.png`

```
投保域
├── 投保流程（5步）
│   ├── 1.提交意向 — 自然语言描述车型+需求
│   ├── 2.解析意图 — 提取品牌/型号/车牌号 → 校验可识别
│   ├── 3.核实身份 — 按用户ID查档案
│   ├── 4.核实车辆 — 车牌→车型→唯一车辆
│   └── 5.风险评估
│       ├── 6因子评分: 年龄+驾龄+事故+车龄+车价, 上限100
│       ├── 保费定价: 车价×2%×风险系数×险种系数
│       └── 三级分流: ≤60→APPROVED | 61-79→REFERRED | ≥80→DECLINED
├── 人工审批 — REFERRED→APPROVED
├── 支付签单 — APPROVED+30天有效期 → 支付 → 签发保单(一年ACTIVE)
└── 状态流转 — 报价单生命周期 + 保单生命周期
```

## 3. 理赔域

> 图片：`doc/diagrams/mindmap-03-理赔域.png`

```
理赔域
├── 理赔流程（5步）
│   ├── 1.提交申请 — 保单号+事故描述+金额
│   ├── 2.保单校验 — 存在+ACTIVE+有效期内
│   ├── 3.事故提取 — 类型/地点/日期/涉及方
│   ├── 4.重复检测 — 同保单同描述→已存在结果
│   └── 5.欺诈评估
│       ├── 4维评分: 理赔/保费比+绝对金额+信息完整+事故类型, 上限100
│       └── 三级分流: <30→APPROVED | 30-69→INVESTIGATING | ≥70→DENIED
├── 人工审核 — INVESTIGATING → 批准(≤保费×5) / 拒绝(0)
└── 状态流转 — PENDING→INVESTIGATING→APPROVED→PAID / →DENIED(终态)
```

## 4. 客服域

> 图片：`doc/diagrams/mindmap-04-客服域.png`

```
客服域
├── 对话流程
│   ├── 用户发送 → 输入安检(注入拦截+话题过滤) → 知识检索(BM25) → 智能回答 → 回复质检
│   └── 知识库: 保险条款 / 理赔指南 / FAQ
└── 会话管理 — 自动创建续期(30分钟过期)，历史20轮截断
```

## 5. 风控护栏

> 图片：`doc/diagrams/mindmap-05-风控护栏.png`

```
风控护栏
├── 输入护栏 — 注入指令 + 提示注入 + 无关话题过滤
├── 业务护栏 — 车辆信息校验(品牌+型号或车牌号)
└── 输出护栏 — 敏感信息检测 + 幻觉检测 + 长度检查
```

典型场景
